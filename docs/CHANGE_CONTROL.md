# Change Control

## Mencegah Scope Creep
Proyek tahap 1 (Versi 1) **telah dikunci** berdasarkan `konsep.md` dan `PROJECT_CHARTER.md`.

Setiap permintaan fitur baru, yang tidak termasuk dalam definisi Modul 1 (Pengajuan) dan Modul 2 (Support) saat ini, harus diperlakukan sebagai **Scope Change**.

### Yang termasuk Scope Change (Ditolak untuk V1):
- Integrasi Single Sign-On (Google Workspace, Active Directory).
- Aplikasi Mobile Native.
- Fitur absensi / integrasi HRIS otomatis.
- *Chatbot* dua arah di WhatsApp (saat ini bot hanya bersifat pemberitahuan / 1-arah).

### Cara Meminta Perubahan Scope:
1. Pihak yang meminta (Stakeholder) mengajukan perubahan ke Manager IT (Pemilik Proyek).
2. Dampak perubahan dianalisis: *Berapa lama penambahan waktu rilisnya?*
3. Jika disetujui, `konsep.md` akan direvisi.
4. Tim Pengembang (AI / Koding) akan memperbarui kode berdasarkan revisi tersebut.
